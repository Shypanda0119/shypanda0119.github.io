Tabletopia

식당 예약, 운영 관리, 지역 기반 추천 기능을 제공하는 레스토랑 플랫폼 백엔드 프로젝트입니다.

Spring Boot Spring MVC JPA MySQL OpenAI API React Admin

프로젝트 소개

  • 식당 예약, 메뉴 관리, 운영시간 관리 기능 제공
  • 관리자 운영 기능과 사용자 추천 기능을 함께 구현
  • 실제 서비스 운영을 고려한 백엔드 구조 설계

담당 역할

  • 관리자 페이지 백엔드 CRUD API 전체 개발
  • 지역 기반 식당 추천 로직 직접 구현
  • OpenAI API 기반 대화형 추천 기능 구현
  • Restaurant, Menu, OpeningHour 등 DB 테이블 구조 설계 참여
  • DTO 기반 조회 구조 설계 및 조회 성능 최적화

핵심 구현 기능

관리자 운영 기능

  • 식당 등록 / 수정 / 비활성화(Soft Delete) API 구현
  • 메뉴, 운영시간, 이미지, 편의시설, 리뷰 관리 기능 개발
  • React Admin에서 사용 가능한 JSON 응답 구조 통일
  • Soft Delete 적용 시 연관 데이터가 노출되지 않도록 조회 조건 통일

지역 기반 식당 추천 기능

  • 사용자 입력 문장에서 “OO구” 형태의 지역 정보 파싱
  • 해당 지역 식당만 조회하는 1차 필터링 로직 구현
  • 카테고리, 평점 기준으로 정렬 후 상위 5개 식당 추천

OpenAI 기반 대화형 추천 기능

  • OpenAI Chat API 기반 메뉴 추천 기능 구현
  • 대화 횟수 5회 제한으로 비용 및 무한 대화 방지
  • API 호출 실패 시 fallback 응답 제공

조회 최적화

  • 목록 및 관리자 화면은 DTO 조회 방식 사용
  • N+1 발생 가능 구간은 조회 전략 분리
  • 이미지 데이터는 바이너리 대신 URL 메타데이터 방식으로 처리

기술적 문제와 해결

운영시간 다중 수정 시 데이터 일관성 문제

  • 운영시간을 여러 건 동시에 수정할 때 일부 데이터만 반영되는 문제 발생 가능
  • 관련 작업을 하나의 트랜잭션으로 묶어 처리
  • 모든 작업이 성공할 때만 데이터 반영되도록 구현

외부 API 의존으로 인한 추천 기능 중단 문제

  • OpenAI API 응답 지연 또는 실패 시 추천 기능이 중단되는 문제 발생
  • fallback 응답 구조 적용
  • API 실패 시 기본 안내 메시지 반환

Soft Delete 데이터 노출 문제

  • 비활성 데이터가 조회 및 추천 결과에 포함될 가능성 존재
  • 공통 조회 조건을 정리해 비활성 데이터 노출 방지

결과

  • 식당 운영 데이터를 관리할 수 있는 관리자 백엔드 시스템 구축
  • 지역 기반 추천 로직과 대화형 추천 기능 결합
  • Soft Delete 기반 데이터 관리 및 조회 일관성 확보
