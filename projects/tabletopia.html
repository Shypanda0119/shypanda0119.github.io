🍽 Tabletopia — 식당 예약 & 관리 플랫폼

Spring Boot Spring MVC JPA MySQL OpenAI API React Admin

📌 프로젝트 개요

식당 예약·웨이팅·메뉴 관리·운영시간 등 식당 운영에 필요한 기능을 제공하는 서비스입니다. 저는 이 프로젝트에서 관리자 백엔드 전체 개발과 지역 기반 추천 로직 · AI 추천 기능을 담당했습니다.

📌 담당 역할 (My Role)

  • 관리자 페이지 백엔드 CRUD 전체 개발
  • 지역 기반 식당 추천 로직 직접 구현
  • OpenAI API 기반 AI 추천 기능 개발
  • DB 테이블 구조 설계 참여
  • JPA 기반 조회 최적화(fech join 가이드 적용)

📌 구현 기능 상세

① 관리자 페이지 CRUD 전체 개발

  • 식당 등록/수정/삭제
  • 메뉴·운영시간·이미지·편의시설·리뷰 관리 API
  • React Admin에서 바로 쓸 수 있도록 JSON 구조 정리
  • 수정/삭제 시 연관 데이터(메뉴·이미지 등) 처리

② 지역 기반 추천 로직 개발

사용자가 "강남 맛집 추천" → Chatbot → 지역 기반 후보 뽑는 구조를 만들었습니다.

  • 주소에서 “OO구” 텍스트만 추출하는 파싱 기능
  • 해당 지역 식당만 필터링하는 1차 로직 직접 구현
  • 카테고리·평점 기준으로 후보 정리

③ ChatGPT 기반 추천 기능

  • OpenAI Chat Completions API 호출
  • 프롬프트 구조 설계 (지역/메뉴/식당 리스트 전달)
  • 응답 중에서 식당명 추출하는 파싱 처리
  • 5회 이내 대화 제공(간단한 대화형 챗봇)
  • 예외 발생 시 fallback 응답 제공

④ JPA 기반 조회 최적화

  • N+1 문제 발생 구간에서 fetch join 사용
  • 필요한 경우 DTO를 사용하여 필요한 값만 조회

⑤ DB 설계 참여

  • Restaurant, Menu, OpeningHour, Image, Facility, Review 테이블 구조 구성
  • 관계(FK) 설정 및 인덱스 설계 일부 참여

📌 기술적 문제 & 해결 과정

  • GPT 응답이 일정하지 않아 정규식으로 식당명 파싱 → 해결
  • React Admin에서 PUT → PATCH 문제 대응을 위해 요청 구조 조정
  • 운영시간 복수 등록 시 트랜잭션 처리 구현

📌 결과

지역 기반 추천 + AI 추천 + 관리자 기능까지 백엔드 핵심 구조를 완성했고, 실제로 작동하는 레스토랑 관리·추천 플랫폼 백엔드를 구축했습니다.
