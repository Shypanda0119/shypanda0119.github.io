🛒 Peach-Store — 전자기기 커머스 플랫폼

Spring MVC JSP Bootstrap MySQL

📌 프로젝트 개요

전자기기를 판매하는 쇼핑몰을 구현한 프로젝트입니다. Apple Store처럼 기종 → 색상 → 용량을 선택하면 해당 옵션에 맞는 상품 정보를 보여주는 구조입니다. 저는 쇼핑몰에서 핵심이 되는 옵션 조합 처리, 상품 상세 데이터 구성, 옵션별 조회 필터링을 맡았습니다.

📌 담당 역할 (My Role)

  • 기종/색상/용량 옵션 조합 구조 설계
  • 옵션 선택 시 상품 정보가 실시간으로 바뀌는 로직 구현
  • 상품 상세페이지 및 옵션별 필터링 구현
  • 상품·옵션 테이블 구조 설계 참여

📌 구현 기능 상세

① 옵션 조합 구조 설계

전자제품은 옵션 조합이 많기 때문에, 먼저 옵션 구조 설계를 맡았습니다.

  • 기종(model) — 예: iPhone 13, iPhone 13 Pro
  • 색상(color) — 예: 블루, 그래파이트
  • 용량(storage) — 예: 128GB, 256GB
  • 위 3가지를 묶어서 하나의 상품 옵션(item)으로 구성
  • 옵션 조합마다 고유 item_id를 부여하는 구조 설계

② 옵션 선택 시 상품 상세정보 동적 변경

  • 사용자가 기종을 선택하면 → 해당 기종의 색상만 표시
  • 색상을 고르면 → 선택 가능한 용량만 표시
  • 마지막 옵션까지 선택하면 → 해당 item 정보(가격/설명/이미지)를 출력
  • JSP + Bootstrap으로 UI 구성

③ 상품 상세 페이지 구현

Apple Store처럼 이미지·설명·가격이 옵션에 따라 바뀌도록 구성했습니다.

  • 상품 소개 이미지 출력
  • 선택된 옵션의 가격 표시
  • 선택 옵션을 URL parameter로 전달하여 새로고침 시에도 옵션 유지

④ 옵션 기반 상품 조회 필터링

  • 옵션 선택 시 DB에서 해당 item_id 조회
  • JSP에서 비동기 요청으로 필요한 값만 불러오기
  • Spring MVC 컨트롤러에서 옵션별 select 처리

📌 DB 설계

쇼핑몰의 옵션 조합이 많기 때문에 구조 정리가 중요했습니다.

  • product(기본 상품 정보)
  • product_option(기종/색상/용량 조합)
  • product_image(옵션별 이미지 저장)
  • item(item_id)로 모든 옵션 조합 관리

📌 기술적 문제 & 해결

  • 옵션 조합이 늘어날 때 중복 옵션이 보여서 필터 순서를 재정의하여 해결
  • 색상/용량 조합이 안 맞는 경우 0건 조회 → "해당 옵션 없음" 추가 안내
  • 상품 상세 페이지 새로고침 시 옵션 초기화 → parameter 유지 방식으로 해결

📌 결과

쇼핑몰에서 가장 중요한 옵션 선택 → 상품 상세정보 출력까지 전체 흐름을 직접 구현해, 전자기기 쇼핑몰의 핵심 기능을 완성했습니다.
