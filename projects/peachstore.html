Peach Store

전자기기 상품을 판매하는 커머스 플랫폼 프로젝트입니다. Apple Store처럼 기종 → 색상 → 용량 옵션 조합을 선택하면 해당 옵션에 맞는 상품 정보를 조회하는 구조로 구현했습니다.

Spring MVC JSP Bootstrap MySQL

프로젝트 소개

  • 전자기기 판매를 위한 커머스 플랫폼 구현
  • 기종, 색상, 용량 조합 기반 상품 조회 구조 설계
  • 옵션 선택에 따라 상품 정보가 변경되는 상세 페이지 구현

담당 역할

  • 기종 / 색상 / 용량 옵션 조합 구조 설계
  • 옵션 선택 기반 상품 상세 페이지 구현
  • 옵션 조합에 따른 상품 조회 및 필터링 로직 구현
  • 상품 및 옵션 관련 테이블 구조 설계 참여

핵심 구현 기능

옵션 조합 구조 설계

  • 기종(model), 색상(color), 용량(storage) 조합 구조 설계
  • 각 옵션 조합을 하나의 item_id로 관리
  • 옵션 조합마다 고유 상품 데이터를 조회할 수 있도록 DB 구조 설계

옵션 선택 기반 상품 상세 페이지

  • 기종 선택 시 해당 기종에 맞는 색상 옵션만 표시
  • 색상 선택 시 선택 가능한 용량 옵션만 표시
  • 모든 옵션 선택 후 해당 item_id 기반 상품 정보 조회
  • 가격, 설명, 이미지가 선택한 옵션에 따라 변경되도록 구현

상품 조회 흐름 구현

  • 선택한 옵션 값을 URL parameter로 전달
  • Spring MVC Controller에서 옵션 조합에 맞는 상품 조회 처리
  • 조회 결과를 기반으로 JSP에서 상품 상세 정보를 다시 렌더링

상품 상세 상태 유지

  • 새로고침 시에도 선택 옵션이 유지되도록 URL parameter 방식 적용
  • 선택 상태가 초기화되지 않도록 상세 페이지 흐름 구성

DB 설계

  • product : 기본 상품 정보
  • product_option : 기종 / 색상 / 용량 조합 정보
  • product_image : 옵션별 상품 이미지
  • 옵션 조합은 item_id 기준으로 관리

기술적 문제와 해결

옵션 조합 중복 표시 문제

  • 옵션 조합 수가 많아지면서 동일한 옵션이 중복 표시되는 문제가 발생
  • 기종 → 색상 → 용량 순으로 필터링 순서를 재정의하여 중복 노출을 줄임

존재하지 않는 옵션 조합 선택 문제

  • 색상과 용량 조합이 맞지 않는 경우 조회 결과가 0건이 되는 상황 발생
  • "해당 옵션 조합 없음" 안내 메시지를 추가해 사용자 혼란을 줄임

상품 상세 페이지 새로고침 시 옵션 초기화 문제

  • 페이지 새로고침 시 선택한 옵션 정보가 초기화되는 문제가 발생
  • 선택 값을 URL parameter로 유지해 동일한 상태가 다시 렌더링되도록 처리

결과

  • 옵션 조합 기반 상품 조회 구조 구현
  • 옵션 선택 → 상품 정보 변경 흐름 구현
  • 전자기기 쇼핑몰의 핵심 기능인 상품 옵션 시스템 완성
