Shinlogis WMS

입고 → 재고 → 출고 / 폐기 흐름을 관리하는 창고 관리 시스템(WMS)입니다. 실제 물류 흐름을 기반으로 재고 상태 관리와 통계 조회 기능을 구현했습니다.

Java Mybatis MySQL Swing

프로젝트 소개

  • 입고 / 재고 / 출고 / 폐기 흐름을 관리하는 창고 관리 시스템 구현
  • 재고 상태와 수량 변화를 중심으로 물류 데이터 흐름 설계
  • Swing 기반 대시보드에서 조회 가능한 통계 API 제공

담당 역할

  • 재고 흐름 처리 로직 구현
  • 유통기한 및 파손 상태 관리 기능 구현
  • 대시보드 통계 API 개발
  • 입고 / 재고 / 출고 / 폐기 관련 DB 구조 설계 참여

핵심 구현 기능

재고 흐름 처리 로직

  • 출고 요청 시 재고 수량 차감 로직 구현
  • 폐기 요청 시 재고 상태 변경 또는 삭제 처리
  • 재고 부족 상황에 대한 예외 처리 추가
  • 입고 → 재고 반영 → 출고까지 이어지는 데이터 흐름 구현

재고 상태 관리

  • 유통기한 정보를 기반으로 재고 상태 분류
  • 유통기한 임박 / 폐기 대상 재고 구분
  • 파손으로 입고된 상품은 별도의 상태로 관리
  • 재고 상태를 enum 구조로 정의해 일관된 데이터 관리

대시보드 통계 API

  • 전체 입고 / 출고 수량 통계 조회
  • 정상 / 임박 / 파손 / 폐기 재고 수량 집계
  • Swing UI에서 조회할 수 있도록 JSON 형태 API 제공

기술적 문제와 해결

동일 상품 재고가 여러 날짜로 누적되는 문제

  • 입고 시점이 다른 동일 상품 재고가 누적되는 구조 발생
  • 출고 시 FIFO 방식으로 먼저 입고된 재고부터 차감하도록 처리

재고 상태 관리 기준 정리

  • 유통기한 / 임박 / 파손 / 폐기 상태 기준이 불명확한 문제 발생
  • 재고 상태를 enum 구조로 정의하여 일관된 상태 관리 적용

대량 조회 시 성능 문제

  • 재고 및 입출고 데이터를 함께 조회할 때 JPA 조회 성능 저하 발생
  • 조회 구조를 정리하고 필요한 데이터만 조회하도록 개선

결과

  • 실제 물류 흐름과 유사한 재고 관리 구조 구현
  • 재고 상태 관리와 FIFO 출고 로직 구현
  • Swing 대시보드와 연동되는 통계 API 개발
