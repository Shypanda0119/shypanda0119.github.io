📦 Shinlogis-WMS — 창고관리 시스템

Spring Boot JPA MySQL Swing

📌 프로젝트 개요

실제 물류 흐름(입고 → 재고 → 출고/폐기)을 기반으로 한 창고 관리 시스템입니다.
저는 재고 흐름 처리 로직과 대시보드 통계 API, 그리고 유통기한/파손 상태 구분 기능을 맡았습니다.

📌 담당 역할 (My Role)

  • 재고 흐름 처리(출고/폐기 시 수량 차감)
  • 유통기한 임박/파손 상태 처리
  • 대시보드 통계 API 개발
  • DB 구조 설계 참여

📌 구현 기능 상세

① 재고 흐름 처리 로직

  • 출고 요청 시 재고 테이블의 수량 차감
  • 폐기 요청 시 해당 재고 항목 삭제 또는 상태 변경
  • 재고가 모자랄 경우 예외 처리
  • 입고 → 재고 반영까지 end-to-end 데이터 흐름 개발

② 유통기한·파손 상태 분류

  • 유통기한이 지나면 자동으로 "폐기 대상"
  • 파손으로 들어온 데이터는 “파손” 상태로 분류

③ 대시보드 통계 API

  • 전체 입/출고 수량 합산
  • 정상/임박/파손/폐기 재고 수량 정리
  • Swing UI에서 조회 가능하도록 JSON 형태로 제공

④ DB 설계

  • 입고/재고/출고/폐기 테이블 설계 참여
  • 재고 상태 enum 구조 설정

📌 기술적 해결 경험

  • 입고한 재고가 여러 날짜로 쌓여 있을 때 FIFO 방식으로 처리
  • 파손/유통기한/임박 구분 논리 정리
  • 대량 데이터 처리 시 JPA fetch 성능 개선

📌 결과

실제 WMS와 유사한 재고 흐름을 구현하며 재고/입출고/유통기한 구조에 대한 이해가 깊어졌고, Swing 대시보드와 연동되는 실용적인 백엔드 API를 개발했습니다.
